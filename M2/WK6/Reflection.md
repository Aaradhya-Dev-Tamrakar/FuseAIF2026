# Reflection — W6 Probabilistic Models & Bayesian Inference

**One concrete example where the fully Bayesian answer would have changed a decision made using only the MLE:**

The VP's second concern — a new contract tier with only 40 enrolled customers — is exactly where MLE-only decisions break down. For **Group A_small (n=40)**, the raw MLE churn rate is **0.3750** (15/40). Read alone, that number says "this segment churns worse than the large Month-to-month group," and a retention team sizing intervention budget off that figure would over-provision for the small segment.

The Beta(2,8)-prior MAP for the same group is **0.3333**, a pull of **|MAP−MLE| = 0.0417** — over 90× larger than the pull for the large groups (Group A large: 0.0006; Group B large: 0.0005). The MLE has no way to signal that this specific estimate is unreliable; it reports 0.3750 with the same apparent confidence it would for a 4,000-customer group. Only the full posterior exposes *why* the MAP diverges so sharply: at n=40, the Beta(2,8) prior still carries real weight, because the prior becomes negligible (posterior mean within 1% of MLE) only around **n≈55–60**. Below that, any single point estimate — MLE or MAP — is presenting false precision.

**Mechanism:** uncertainty quantification. A p-value or MLE collapses the segment to one number. The 94% HDI around the small-segment posterior communicates the actual reliability of that number, which is what the VP's question ("how certain are you?") was really asking. Reporting "37.5% churn" to the VP is a materially different decision-input than reporting "37.5%, but with wide uncertainty at n=40, and the point estimate itself would still be moving with each new customer we onboard" — the first invites a budget commitment; the second invites a wait-and-collect-more-data recommendation, which is the correct one at this sample size.

The same pattern shows up in Part 6: the Bayesian 94% HDI for β_Contract_Month-to-month ([1.0845, 1.4920], mean 1.2932) and the frequentist MLE (1.2927) land on nearly the same point estimate — but only the HDI supports the direct probability statement "P(β ∈ [1.0845, 1.4920] | data) = 0.94." A frequentist CI licenses no such claim about this specific interval; it's a statement about the long-run procedure, not this data. When a decision needs to be defended as "we are 94% confident the effect lies in this range," only the Bayesian framework actually earns that sentence.
